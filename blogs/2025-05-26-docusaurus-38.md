<p>We are happy to announce <strong>Docusaurus 3.8</strong>.</p>
<p>This release improves build performance, includes new features and introduces "Future Flags" to prepare your site for Docusaurus 4.</p>
<p>Upgrading is easy. We follow <a class="" href="https://semver.org/" rel="noopener noreferrer" target="_blank">Semantic Versioning</a>, and minor version updates have <strong>no breaking changes</strong>, accordingly to our <a class="" href="https://docusaurus.io/community/release-process">release process</a>.</p>

<h2 class="anchor anchorTargetHideOnScrollNavbar_qJOx" id="performance">Performance<a class="hash-link" href="https://docusaurus.io/blog/releases/3.8#performance" title="Direct link to Performance">​</a></h2>

<h4 class="anchor anchorTargetHideOnScrollNavbar_qJOx" id="persistent-cache">Persistent Cache<a class="hash-link" href="https://docusaurus.io/blog/releases/3.8#persistent-cache" title="Direct link to Persistent Cache">​</a></h4>
<p>In <a class="" href="https://github.com/facebook/docusaurus/pull/10931" rel="noopener noreferrer" target="_blank">#10931</a>, we have enabled the <strong><a class="" href="https://rspack.dev/blog/announcing-1-2#persistent-cache" rel="noopener noreferrer" target="_blank">Rspack persistent cache</a></strong>. Similarly to the <a class="" href="https://webpack.js.org/blog/2020-10-10-webpack-5-release/#persistent-caching" rel="noopener noreferrer" target="_blank">Webpack persistent cache</a> (already supported), it permits to greatly speed up the bundling of the Docusaurus app on subsequent builds.</p>
<p>In practice, your site should build much faster if you run <code>docusaurus build</code> a second time.</p>
<p>This feature depends on using the <a class="" href="https://rspack.dev/" rel="noopener noreferrer" target="_blank">Rspack bundler</a>, and can be turned on with the <code>rspackPersistentCache</code> flag:</p>

<h4 class="anchor anchorTargetHideOnScrollNavbar_qJOx" id="worker-threads">Worker Threads<a class="hash-link" href="https://docusaurus.io/blog/releases/3.8#worker-threads" title="Direct link to Worker Threads">​</a></h4>
<p>In <a class="" href="https://github.com/facebook/docusaurus/pull/10826" rel="noopener noreferrer" target="_blank">#10826</a>, we introduced a <a class="" href="https://github.com/tinylibs/tinypool" rel="noopener noreferrer" target="_blank">Node.js Worker Thread pool</a> to run the static side generation code. With this new strategy, we can better leverage all the available CPUs, reduce static site generation time, and contain potential memory leaks.</p>
<p>This feature can be turned on with the <code>ssgWorkerThreads</code> flag, and requires the <a class="" href="https://docusaurus.io/blog/releases/3.8#postbuild-change"><code>v4.removeLegacyPostBuildHeadAttribute</code></a> Future Flag to be turned on:</p>

<h3 class="anchor anchorTargetHideOnScrollNavbar_qJOx" id="other-optimizations">Other Optimizations<a class="hash-link" href="https://docusaurus.io/blog/releases/3.8#other-optimizations" title="Direct link to Other Optimizations">​</a></h3>
<p>We identified and resolved several major bottlenecks, including:</p>
<ul>
<li class="">In <a class="" href="https://github.com/facebook/docusaurus/pull/11007" rel="noopener noreferrer" target="_blank">#11007</a>, we optimized the dev server startup time for macOS users. We figured out that the code to open your browser used expensive synchronous/blocking calls that prevented the bundler from doing its work. From now on, the bundler and the macOS browser opening will run in parallel, leading to a faster <code>docusaurus start</code> experience for all macOS users.</li>
<li class="">In <a class="" href="https://github.com/facebook/docusaurus/pull/11163" rel="noopener noreferrer" target="_blank">#11163</a>, we noticed that the docs <code>showLastUpdateAuthor</code> and <code>showLastUpdateTime</code> are quite expensive, and require to run a <code>git log</code> command for each document. On large sites, running these commands in parallel can exhaust the system and lead to Node.js <code>EBADF</code> errors. We implemented a Git command queue to avoid exhausting the system, which also slightly increased performance of our plugin's <code>loadContent()</code> lifecycle.</li>
<li class="">In <a class="" href="https://github.com/facebook/docusaurus/pull/10885" rel="noopener noreferrer" target="_blank">#10885</a>, we implemented an SVG sprite for the external link icon. Due to its repeated appearance in various places of your site (navbar, footer, sidebars...), using a React SVG component lead to duplicated SVG markup in the final HTML. Using a sprite permits to only embed the icon SVG once, reducing the generated HTML size and the static generation time. We <a class="" href="https://github.com/facebook/docusaurus/issues/5865" rel="noopener noreferrer" target="_blank">plan to use more SVG sprites</a> in the future.</li>
<li class="">In <a class="" href="https://github.com/facebook/docusaurus/pull/11176" rel="noopener noreferrer" target="_blank">#11176</a>, we fine-tuned the webpack/Rspack configuration to remove useless optimizations.</li>
</ul>

<h2 class="anchor anchorTargetHideOnScrollNavbar_qJOx" id="future-flags">Future Flags<a class="hash-link" href="https://docusaurus.io/blog/releases/3.8#future-flags" title="Direct link to Future Flags">​</a></h2>

<h3 class="anchor anchorTargetHideOnScrollNavbar_qJOx" id="css-cascade-layers">CSS Cascade Layers<a class="hash-link" href="https://docusaurus.io/blog/releases/3.8#css-cascade-layers" title="Direct link to CSS Cascade Layers">​</a></h3>
<p>In Docusaurus v4, we plan to leverage <a class="" href="https://css-tricks.com/css-cascade-layers/" rel="noopener noreferrer" target="_blank">CSS Cascade Layers</a>. This modern CSS feature is widely supported and permits to group CSS rules in layers of specificity. It is particularly useful to give you more control over the CSS cascade. It makes the CSS rules less dependent on their insertion order. Your un-layered rules will now always override the layered CSS we provide.</p>
<p>In <a class="" href="https://github.com/facebook/docusaurus/pull/11142" rel="noopener noreferrer" target="_blank">#11142</a>, we implemented a new experimental <a class="" href="https://docusaurus.io/docs/api/plugins/@docusaurus/plugin-css-cascade-layers"><code>@docusaurus/plugin-css-cascade-layers</code></a> that you can turn on through the <code>v4.useCssCascadeLayers</code> flag if you use the classic preset:</p>

<h2 class="anchor anchorTargetHideOnScrollNavbar_qJOx" id="maintenance">Maintenance<a class="hash-link" href="https://docusaurus.io/blog/releases/3.8#maintenance" title="Direct link to Maintenance">​</a></h2>
<p>Docusaurus 3.8 is ready for <a class="" href="https://github.com/facebook/docusaurus/pull/11168" rel="noopener noreferrer" target="_blank">Node.js 24</a> and <a class="" href="https://github.com/facebook/docusaurus/pull/10966" rel="noopener noreferrer" target="_blank">TypeScript 5.8</a>.</p>
<p>We also removed useless npm packages and internalizing unmaintained ones:</p>
<ul>
<li class="">In <a class="" href="https://github.com/facebook/docusaurus/pull/11010" rel="noopener noreferrer" target="_blank">#11010</a> and <a class="" href="https://github.com/facebook/docusaurus/pull/11014" rel="noopener noreferrer" target="_blank">#11014</a>, we internalized the unmaintained <code>react-ideal-image</code> and <code>react-waypoint</code> package used in <code>@docusaurus/plugin-ideal-image</code>, and made them compatible with React 19.</li>
<li class="">In <a class="" href="https://github.com/facebook/docusaurus/pull/10956" rel="noopener noreferrer" target="_blank">#10956</a>, we removed our dependency to the unmaintained <code>react-dev-utils</code> package (from Create-React-App).</li>
<li class="">In <a class="" href="https://github.com/facebook/docusaurus/pull/10358" rel="noopener noreferrer" target="_blank">#10358</a>, we replaced the unmaintained <code>shelljs</code> package by <code>execa</code></li>
<li class="">In <a class="" href="https://github.com/facebook/docusaurus/pull/11138" rel="noopener noreferrer" target="_blank">#11138</a>, we removed the <code>reading-time</code> package in favor of the built-in <code>Intl.Segmenter</code> standard API to compute blog post reading times.</li>
<li class="">In <a class="" href="https://github.com/facebook/docusaurus/pull/11037" rel="noopener noreferrer" target="_blank">#11037</a>, we removed the useless <code>clean-webpack-plugin</code>.</li>
</ul>
<h2 class="anchor anchorTargetHideOnScrollNavbar_qJOx" id="other-changes">Other changes<a class="hash-link" href="https://docusaurus.io/blog/releases/3.8#other-changes" title="Direct link to Other changes">​</a></h2>
<p>Other notable changes include:</p>
<ul>
<li class="">In <a class="" href="https://github.com/facebook/docusaurus/pull/10852" rel="noopener noreferrer" target="_blank">#10852</a>, the theme <code>docsVersionDropdown</code> navbar item now accepts a <code>versions</code> attribute.</li>
<li class="">In <a class="" href="https://github.com/facebook/docusaurus/pull/10953" rel="noopener noreferrer" target="_blank">#10953</a>, the <code>@docusaurus/remark-plugin-npm2yarn</code> plugin now supports Bun tabs conversions by default.</li>
<li class="">In <a class="" href="https://github.com/facebook/docusaurus/pull/10945" rel="noopener noreferrer" target="_blank">#10945</a>, more stable CSS classes are now applied to the main theme layout elements, to let you create more reliable CSS selectors.</li>
<li class="">In <a class="" href="https://github.com/facebook/docusaurus/pull/10846" rel="noopener noreferrer" target="_blank">#10846</a>, the Markdown code block <code>showLineNumbers</code> metastring can now accept a number to initialize the line counter initial value.</li>
<li class="">In <a class="" href="https://github.com/facebook/docusaurus/pull/11090" rel="noopener noreferrer" target="_blank">#11090</a>, we made it easier to provide a custom page title formatter.</li>
<li class="">In <a class="" href="https://github.com/facebook/docusaurus/pull/11088" rel="noopener noreferrer" target="_blank">#11088</a>, the page plugin now supports <code>frontMatter.slug</code> like docs and blog plugins already do.</li>
<li class="">In <a class="" href="https://github.com/facebook/docusaurus/pull/10875" rel="noopener noreferrer" target="_blank">#10875</a>, the docs versioning CLI now also copies localized docs JSON translation files.</li>
</ul>
