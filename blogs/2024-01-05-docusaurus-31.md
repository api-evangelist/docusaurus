<p>We are happy to announce <strong>Docusaurus 3.1</strong>.</p>
<p>The upgrade should be easy: as explained in our <a class="" href="https://docusaurus.io/community/release-process">release process documentation</a>, minor versions respect <a class="" href="https://semver.org/" rel="noopener noreferrer" target="_blank">Semantic Versioning</a>.</p>

<h2 class="anchor anchorTargetHideOnScrollNavbar_qJOx" id="highlights">Highlights<a class="hash-link" href="https://docusaurus.io/blog/releases/3.1#highlights" title="Direct link to Highlights">​</a></h2>
<h3 class="anchor anchorTargetHideOnScrollNavbar_qJOx" id="broken-anchors-checker">Broken anchors checker<a class="hash-link" href="https://docusaurus.io/blog/releases/3.1#broken-anchors-checker" title="Direct link to Broken anchors checker">​</a></h3>
<p>In <a class="" href="https://github.com/facebook/docusaurus/pull/9528" rel="noopener noreferrer" target="_blank">#9528</a>, we improved the built-in broken links checker to also detect broken anchors.</p>
<p><img alt="Docusaurus blog post social card" class="img_vXGZ" height="896" src="https://docusaurus.io/assets/images/broken-anchor-4191e5dd94aef9e8c5e3524880670f0f.jpg" width="1920" /></p>
<div class="theme-admonition theme-admonition-tip admonition_o5H7 alert alert--success"><div class="admonitionHeading_FzoX"><span class="admonitionIcon_rXq6"><svg viewBox="0 0 12 16" xmlns="http://www.w3.org/2000/svg"><path d="M6.5 0C3.48 0 1 2.19 1 5c0 .92.55 2.25 1 3 1.34 2.25 1.78 2.78 2 4v1h5v-1c.22-1.22.66-1.75 2-4 .45-.75 1-2.08 1-3 0-2.81-2.48-5-5.5-5zm3.64 7.48c-.25.44-.47.8-.67 1.11-.86 1.41-1.25 2.06-1.45 3.23-.02.05-.02.11-.02.17H5c0-.06 0-.13-.02-.17-.2-1.17-.59-1.83-1.45-3.23-.2-.31-.42-.67-.67-1.11C2.44 6.78 2 5.65 2 5c0-2.2 2.02-4 4.5-4 1.22 0 2.36.42 3.22 1.19C10.55 2.94 11 3.94 11 5c0 .66-.44 1.78-.86 2.48zM4 14h5c-.23 1.14-1.3 2-2.5 2s-2.27-.86-2.5-2z" fill-rule="evenodd"></path></svg></span>Make it fail fast</div><div class="admonitionContent_Knsx"><p>The new <a class="" href="https://docusaurus.io/docs/api/docusaurus-config#onBrokenAnchors"><code>onBrokenAnchors</code></a> option has value <code>warn</code> by default, for retro-compatibility reasons.</p><p>We recommend to turn it to <code>throw</code> and fail your CI builds instead of deploying broken anchors to productions.</p></div></div>
<div class="theme-admonition theme-admonition-note admonition_o5H7 alert alert--secondary"><div class="admonitionHeading_FzoX"><span class="admonitionIcon_rXq6"><svg viewBox="0 0 14 16" xmlns="http://www.w3.org/2000/svg"><path d="M6.3 5.69a.942.942 0 0 1-.28-.7c0-.28.09-.52.28-.7.19-.18.42-.28.7-.28.28 0 .52.09.7.28.18.19.28.42.28.7 0 .28-.09.52-.28.7a1 1 0 0 1-.7.3c-.28 0-.52-.11-.7-.3zM8 7.99c-.02-.25-.11-.48-.31-.69-.2-.19-.42-.3-.69-.31H6c-.27.02-.48.13-.69.31-.2.2-.3.44-.31.69h1v3c.02.27.11.5.31.69.2.2.42.31.69.31h1c.27 0 .48-.11.69-.31.2-.19.3-.42.31-.69H8V7.98v.01zM7 2.3c-3.14 0-5.7 2.54-5.7 5.68 0 3.14 2.56 5.7 5.7 5.7s5.7-2.55 5.7-5.7c0-3.15-2.56-5.69-5.7-5.69v.01zM7 .98c3.86 0 7 3.14 7 7s-3.14 7-7 7-7-3.12-7-7 3.14-7 7-7z" fill-rule="evenodd"></path></svg></span>note</div><div class="admonitionContent_Knsx"><p>For users and plugin authors implementing custom <code>&lt;Heading&gt;</code> and <code>&lt;Link&gt;</code> components, we provide a new <a class="" href="https://docusaurus.io/docs/docusaurus-core#useBrokenLinks"><code>useBrokenLinks</code></a> React hook API.</p><p><strong>Most Docusaurus users don't need to care about it</strong>, built-in components (<code>docusaurus/Link</code> and <code>@theme/Heading</code>) already use it internally.</p></div></div>
<h3 class="anchor anchorTargetHideOnScrollNavbar_qJOx" id="parsefrontmatter-hook"><code>parseFrontMatter</code> hook<a class="hash-link" href="https://docusaurus.io/blog/releases/3.1#parsefrontmatter-hook" title="Direct link to parsefrontmatter-hook">​</a></h3>
<p>In <a class="" href="https://github.com/facebook/docusaurus/pull/9624" rel="noopener noreferrer" target="_blank">#9624</a>, we added a new <a class="" href="https://docusaurus.io/docs/api/docusaurus-config#markdown"><code>siteConfig.markdown.parseFrontMatter</code> function hook</a>.</p>
<p>This makes it possible to implement convenient front matter transformations, shortcuts, or to integrate with external systems using front matter that Docusaurus plugins do not support.</p>
