<p>We are happy to announce <strong>Docusaurus 3.9</strong>.</p>
<p>This release drops support for Node.js 18, adds support for Algolia DocSearch v4 with AskAI, improves i18n support, adds Mermaid ELK layout support, and comes with various other improvements and bug fixes.</p>
<p>Upgrading is easy. We follow <a class="" href="https://semver.org/" rel="noopener noreferrer" target="_blank">Semantic Versioning</a>, and minor version updates have <strong>no breaking changes</strong>, accordingly to our <a class="" href="https://docusaurus.io/community/release-process">release process</a>. Note that <strong>we consider dropping End-of-Life Node.js versions as non-breaking changes</strong>.</p>

<h2 class="anchor anchorTargetHideOnScrollNavbar_qJOx" id="dropping-nodejs-18">Dropping Node.js 18<a class="hash-link" href="https://docusaurus.io/blog/releases/3.9#dropping-nodejs-18" title="Direct link to Dropping Node.js 18">​</a></h2>
<p>In <a class="" href="https://github.com/facebook/docusaurus/pull/11408" rel="noopener noreferrer" target="_blank">#11408</a>, we have dropped support for Node.js 18, and the new minimum required Node.js version is now v20.0.</p>
<p>Although it may look like a runtime requirement breaking change, Node.js 18 reached <a class="" href="https://nodejs.org/en/about/releases/" rel="noopener noreferrer" target="_blank">End-of-Life</a>. It <a class="" href="https://nodejs.org/en/blog/announcements/node-18-eol-support" rel="noopener noreferrer" target="_blank">won't receive security updates</a>, and you shouldn't use it anymore. Dropping End-of-Life versions of Node.js on minor version releases is a common practice in the Node.js ecosystem, that we now officially endorse and document on our <a class="" href="https://docusaurus.io/community/release-process#nodejs-support">release process</a>.</p>
<p>This upgrade is further motivated by our dependencies:</p>
<ul>
<li class="">Some of our dependencies now only receive security patches on newer versions that do not support Node.js 18 anymore. See, for example, this <a class="" href="https://github.com/facebook/docusaurus/pull/11410" rel="noopener noreferrer" target="_blank"><code>webpack-dev-server@4</code> security warning</a>. To keep the Docusaurus v3 release line secure for our users, for both the runtime and third-party dependencies, the upgrade is necessary.</li>
<li class="">Some of our dependencies are also dropping support for Node.js 18 in minor releases, transitively impacting Docusaurus runtime requirements. For example, <a class="" href="https://rspack.rs/blog/announcing-1-5" rel="noopener noreferrer" target="_blank">Rspack 1.5 now requires Node.js 18.12</a>, while Docusaurus v3 initially supported Node.js 18.0.</li>
</ul>
<h2 class="anchor anchorTargetHideOnScrollNavbar_qJOx" id="docsearch-v4">DocSearch v4<a class="hash-link" href="https://docusaurus.io/blog/releases/3.9#docsearch-v4" title="Direct link to DocSearch v4">​</a></h2>

<h2 class="anchor anchorTargetHideOnScrollNavbar_qJOx" id="mermaid-elk-layouts">Mermaid ELK layouts<a class="hash-link" href="https://docusaurus.io/blog/releases/3.9#mermaid-elk-layouts" title="Direct link to Mermaid ELK layouts">​</a></h2>

<h2 class="anchor anchorTargetHideOnScrollNavbar_qJOx" id="other-changes">Other changes<a class="hash-link" href="https://docusaurus.io/blog/releases/3.9#other-changes" title="Direct link to Other changes">​</a></h2>
<p>Other notable changes include:</p>
<ul>
<li class="">In <a class="" href="https://github.com/facebook/docusaurus/pull/11283" rel="noopener noreferrer" target="_blank">#11283</a>, we added <code>siteConfig.markdown.hooks.{onBrokenMarkdownLinks,onBrokenMarkdownImages}</code> and deprecated <code>siteConfig.onBrokenMarkdownLinks</code>. The new callback hooks also let you recover from broken links/images by returning a fallback URL.</li>
<li class="">In <a class="" href="https://github.com/facebook/docusaurus/pull/11282" rel="noopener noreferrer" target="_blank">#11282</a>, we added the <code>siteConfig.markdown.emoji</code> config option to disable the previously hard-coded <code>remark-emoji</code> behavior.</li>
<li class="">In <a class="" href="https://github.com/facebook/docusaurus/pull/11397" rel="noopener noreferrer" target="_blank">#11397</a>, we can now resolve site-aliased Markdown links starting with <code>@site/*</code>, that we already supported for ES imports.</li>
<li class="">In <a class="" href="https://github.com/facebook/docusaurus/pull/11294" rel="noopener noreferrer" target="_blank">#11294</a> and <a class="" href="https://github.com/facebook/docusaurus/pull/11415" rel="noopener noreferrer" target="_blank">#11415</a>, we upgraded to Rspack 1.5 and leveraging new Rspack config options to make Docusaurus build faster.</li>
<li class="">In <a class="" href="https://github.com/facebook/docusaurus/pull/11356" rel="noopener noreferrer" target="_blank">#11356</a>, we improved the display of docs sidebar items with long labels.</li>
<li class="">In <a class="" href="https://github.com/facebook/docusaurus/pull/11405" rel="noopener noreferrer" target="_blank">#11405</a>, we improved visual glitches caused by the <code>useColorMode()</code> hook when switching color modes, glitches that also affected your site's logo in the navbar.</li>
