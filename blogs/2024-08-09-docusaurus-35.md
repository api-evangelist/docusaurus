<p>We are happy to announce <strong>Docusaurus 3.5</strong>.</p>
<p>This release contains many <strong>new exciting blog features</strong>.</p>
<p>Upgrading should be easy. Our <a class="" href="https://docusaurus.io/community/release-process">release process</a> respects <a class="" href="https://semver.org/" rel="noopener noreferrer" target="_blank">Semantic Versioning</a>. Minor versions do not include any breaking changes.</p>

<h3 class="anchor anchorTargetHideOnScrollNavbar_qJOx" id="blog-sidebar-grouping">Blog Sidebar Grouping<a class="hash-link" href="https://docusaurus.io/blog/releases/3.5#blog-sidebar-grouping" title="Direct link to Blog Sidebar Grouping">​</a></h3>
<p>In <a class="" href="https://github.com/facebook/docusaurus/pull/10252" rel="noopener noreferrer" target="_blank">#10252</a>, we added support for grouping blog posts by years in the blog sidebar.</p>
<p><img alt="Screenshot of the Docusaurus blog, in particular the sidebar items grouped by year" class="img_vXGZ" height="452" src="https://docusaurus.io/assets/images/blog-sidebar-years-1d0be02836d9a3307386c365b9969a37.png" width="910" /></p>
<p>This feature is now turned on by default, but can be disabled with <code>themeConfig.blog.sidebar.groupByYear: false</code>.</p>
<h3 class="anchor anchorTargetHideOnScrollNavbar_qJOx" id="blog-consistency-options">Blog Consistency Options<a class="hash-link" href="https://docusaurus.io/blog/releases/3.5#blog-consistency-options" title="Direct link to Blog Consistency Options">​</a></h3>
<p>We added new blog options to enforce recommended practices for your blog posts:</p>
<h4 class="anchor anchorTargetHideOnScrollNavbar_qJOx" id="oninlineauthors"><code>onInlineAuthors</code><a class="hash-link" href="https://docusaurus.io/blog/releases/3.5#oninlineauthors" title="Direct link to oninlineauthors">​</a></h4>
<p>We believe large multi-blogs are easier to manage by using <a class="" href="https://docusaurus.io/docs/blog#global-authors">global authors</a>, declared in <code>authors.yml</code>. This notably permits to avoids duplicating author information across multiple blog posts, and now permits to generate <a class="" href="https://docusaurus.io/docs/blog#authors-pages">author pages</a>.</p>
<p>In <a class="" href="https://github.com/facebook/docusaurus/pull/10224" rel="noopener noreferrer" target="_blank">#10224</a>, we added the <code>onInlineAuthors</code> option. Use <code>onInlineAuthors: 'throw'</code> to forbid <a class="" href="https://docusaurus.io/docs/blog#inline-authors">inline authors</a>, and enforce a consistent usage of <a class="" href="https://docusaurus.io/docs/blog#global-authors">global authors</a>.</p>
<h4 class="anchor anchorTargetHideOnScrollNavbar_qJOx" id="onuntruncatedblogposts"><code>onUntruncatedBlogPosts</code><a class="hash-link" href="https://docusaurus.io/blog/releases/3.5#onuntruncatedblogposts" title="Direct link to onuntruncatedblogposts">​</a></h4>
<p>We believe blog posts are better using <a class="" href="https://docusaurus.io/docs/blog#blog-list">truncation markers</a> (<code>&lt;!-- truncate --&gt;</code> or <code>{/* truncate */}</code>). On paginated lists (blog home, tags pages, authors pages), this permits to render a more concise preview of the blog post instead of a full blog post.</p>
<p>In <a class="" href="https://github.com/facebook/docusaurus/pull/10375" rel="noopener noreferrer" target="_blank">#10375</a>, we added the <code>onUntruncatedBlogPosts</code> option. Use <code>onUntruncatedBlogPosts: 'throw'</code> to enforce a consistent usage of <a class="" href="https://docusaurus.io/docs/blog#blog-list">truncation markers</a>.</p>
