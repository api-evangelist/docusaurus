<p>We are happy to announce <strong>Docusaurus 3.6</strong>.</p>
<p>Docusaurus is now ⚡️⚡️⚡️ <strong>much faster</strong> to build your site.</p>
<p>Upgrading should be easy. Our <a class="" href="https://docusaurus.io/community/release-process">release process</a> respects <a class="" href="https://semver.org/" rel="noopener noreferrer" target="_blank">Semantic Versioning</a>. Minor versions do not include any breaking changes.</p>
<p><img alt="Docusaurus blog post social card" class="img_vXGZ" height="600" src="https://docusaurus.io/assets/images/social-card-5214adcb56c496fa6afc571fb4d378f6.png" width="1200" /></p>
<!-- -->
<h2 class="anchor anchorTargetHideOnScrollNavbar_qJOx" id="highlights">Highlights<a class="hash-link" href="https://docusaurus.io/blog/releases/3.6#highlights" title="Direct link to Highlights">​</a></h2>
<p>This release has been mostly focused on build performance through the <a class="" href="https://github.com/facebook/docusaurus/issues/10556" rel="noopener noreferrer" target="_blank">Docusaurus Faster</a> project.</p>
<h2 class="anchor anchorTargetHideOnScrollNavbar_qJOx" id="docusaurus-faster">Docusaurus Faster<a class="hash-link" href="https://docusaurus.io/blog/releases/3.6#docusaurus-faster" title="Direct link to Docusaurus Faster">​</a></h2>
<p>The <a class="" href="https://github.com/facebook/docusaurus/issues/10556" rel="noopener noreferrer" target="_blank">Docusaurus Faster</a> project's goal is to reduce the build times and memory consumption.</p>
<p>We have worked on multiple optimizations and modernized our infrastructure to use faster Rust-based tools, notably:</p>
<ul>
<li class="">🦀 <a class="" href="https://rspack.dev/" rel="noopener noreferrer" target="_blank">Rspack</a>: Fast Rust-based web bundler, almost drop-in replacement for <a class="" href="https://webpack.js.org/" rel="noopener noreferrer" target="_blank">webpack</a></li>
<li class="">🦀 <a class="" href="https://swc.rs/" rel="noopener noreferrer" target="_blank">SWC</a>: Speedy Web Compiler, Rust-based platform for the Web (HTML, CSS, JS)</li>
<li class="">🦀 <a class="" href="https://lightningcss.dev/" rel="noopener noreferrer" target="_blank">Lightning CSS</a>: An extremely fast CSS parser, transformer, bundler, and minifier</li>
</ul>
<h3 class="anchor anchorTargetHideOnScrollNavbar_qJOx" id="impacts">Impacts<a class="hash-link" href="https://docusaurus.io/blog/releases/3.6#impacts" title="Direct link to Impacts">​</a></h3>
<p>Adopting a new infrastructure can have various impacts. It's impossible to list them all exhaustively, so let's focus on the major ones.</p>
<p>To help you adopt it easily, we have been fairly conservative in terms of expected static site output and <a class="" href="https://docusaurus.io/docs/browser-support">browser support</a>.</p>
<h4 class="anchor anchorTargetHideOnScrollNavbar_qJOx" id="build-time">Build Time<a class="hash-link" href="https://docusaurus.io/blog/releases/3.6#build-time" title="Direct link to Build Time">​</a></h4>
<p>Benchmarks on community site show that you can expect your production site to build ⚡️<strong>2 to 4 times faster</strong>! 🔥:</p>
<ul>
<li class=""><a class="" href="https://github.com/facebook/react-native-website/pull/4268" rel="noopener noreferrer" target="_blank">React Native website</a>: 3.04x faster 🔥</li>
<li class=""><a class="" href="https://github.com/babel/website/pull/2997" rel="noopener noreferrer" target="_blank">Babel website</a>: 3.27x faster 🔥</li>
<li class=""><a class="" href="https://github.com/facebook/lexical/pull/6761" rel="noopener noreferrer" target="_blank">Lexical website</a>: 2x faster 🔥</li>
</ul>

<h4 class="anchor anchorTargetHideOnScrollNavbar_qJOx" id="memory-consumption">Memory Consumption<a class="hash-link" href="https://docusaurus.io/blog/releases/3.6#memory-consumption" title="Direct link to Memory Consumption">​</a></h4>
<p>You should also notice an improvement in memory consumption:</p>
<ul>
<li class="">The new infrastructure consumes less memory overall</li>
<li class="">We fixed an important <a class="" href="https://github.com/facebook/docusaurus/pull/10599" rel="noopener noreferrer" target="_blank">memory leak</a> that affects in particular i18n sites</li>
<li class="">We added <a class="" href="https://github.com/facebook/docusaurus/pull/10601" rel="noopener noreferrer" target="_blank">CI checks</a> to ensure that we don't regress, and that our site and init template can both build in a memory-constrained environments</li>
<li class="">We added <a class="" href="https://github.com/facebook/docusaurus/pull/10590" rel="noopener noreferrer" target="_blank">internal tooling</a> to better understand which step of a build consumes memory</li>
<li class="">We <a class="" href="https://github.com/facebook/docusaurus/pull/10410" rel="noopener noreferrer" target="_blank">removed a <code>process.exit(0)</code></a> that can hide memory leaks in your own code and third-party plugins</li>
</ul>
<h4 class="anchor anchorTargetHideOnScrollNavbar_qJOx" id="other-impacts">Other Impacts<a class="hash-link" href="https://docusaurus.io/blog/releases/3.6#other-impacts" title="Direct link to Other Impacts">​</a></h4>
<ul>
<li class="">The HTML minifier now <a class="" href="https://github.com/facebook/docusaurus/discussions/10580" rel="noopener noreferrer" target="_blank">emits warnings in case of invalid HTML markup</a></li>
<li class="">The static HTML output is <a class="" href="https://github.com/facebook/docusaurus/pull/10554" rel="noopener noreferrer" target="_blank">~5% smaller</a></li>
<li class="">Some bugs are automatically fixed by the new infrastructure</li>
</ul>
<h3 class="anchor anchorTargetHideOnScrollNavbar_qJOx" id="adoption-strategy">Adoption Strategy<a class="hash-link" href="https://docusaurus.io/blog/releases/3.6#adoption-strategy" title="Direct link to Adoption Strategy">​</a></h3>

<h4 class="anchor anchorTargetHideOnScrollNavbar_qJOx" id="plugins">Plugins<a class="hash-link" href="https://docusaurus.io/blog/releases/3.6#plugins" title="Direct link to Plugins">​</a></h4>
<p>The new infrastructure uses <a class="" href="https://rspack.dev/" rel="noopener noreferrer" target="_blank">Rspack</a>. By chance, Rspack is almost 100% compatible with <a class="" href="https://webpack.js.org/" rel="noopener noreferrer" target="_blank">webpack</a>, and Rspack shouldn't break our plugin ecosystem.</p>
<p>Most Docusaurus plugins should work out of the box with Rspack, even those implementing <a class="" href="https://docusaurus.io/docs/api/plugin-methods/lifecycle-apis#configureWebpack"><code>configureWebpack</code></a>.</p>

<h2 class="anchor anchorTargetHideOnScrollNavbar_qJOx" id="rsdoctor-plugin">Rsdoctor plugin<a class="hash-link" href="https://docusaurus.io/blog/releases/3.6#rsdoctor-plugin" title="Direct link to Rsdoctor plugin">​</a></h2>
